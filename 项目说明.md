# 📦 OpenClaw 部署门户 - 项目说明

## 🎯 这个项目是什么？

这是一个专为 **Windows + Docker Desktop** 环境设计的 OpenClaw 一键部署门户。它提供了一个 Web 界面，让您可以在浏览器中观看自动化部署过程，而不需要在命令行中操作。

## 🌟 主要特点

1. **Windows 专用** - 针对 Docker Desktop 优化
2. **一键启动** - 双击批处理文件即可运行
3. **可视化界面** - 实时终端输出 + 进度条
4. **扫码配置** - 自动提示飞书二维码扫描
5. **多租户支持** - 可以为不同客户创建独立配置

## 🚀 使用步骤

### 第一次使用

1. **确认 Docker Desktop 运行**
   - 查看系统托盘 Docker 图标
   - 确认显示"运行中"

2. **确认有 OpenClaw 镜像**
   ```powershell
   docker images | findstr openclaw
   ```

3. **双击运行检查**
   ```
   检查环境.bat
   ```

4. **如果检查通过，启动应用**
   ```
   启动应用.bat
   ```

5. **打开浏览器**
   ```
   http://localhost:3000
   ```

6. **点击"开始部署"**
   - 观察终端输出
   - 等待进度到 80%
   - 使用飞书 App 扫描二维码
   - 等待配置完成

### 日常使用

只需双击 `启动应用.bat` 即可！

## 🔧 配置说明

### 主要配置项（.env 文件）

```env
# 端口配置
PORT=3001                    # 后端端口
CLIENT_PORT=3000             # 前端端口

# Docker 配置
DOCKER_SOCK=//./pipe/docker_engine  # Windows Docker Desktop 连接

# OpenClaw 镜像（最重要！）
OPENCLAW_IMAGE=openclaw:latest      # 您的镜像名称

# 多租户配置
TENANT_ID=default                    # 租户ID
CONFIG_BASE_PATH=C:\Users\...\openclaw-configs  # 配置存储路径
```

### 如何修改配置

1. 找到您的 OpenClaw 镜像名称：
   ```powershell
   docker images | findstr openclaw
   ```

2. 编辑 `.env` 文件：
   ```powershell
   notepad .env
   ```

3. 修改 `OPENCLAW_IMAGE` 为您的镜像名称

## 🔍 故障排除

### 问题1：Docker 连接失败

**检查：**
1. Docker Desktop 是否运行？
2. 系统托盘 Docker 图标状态

**解决：**
```
启动 Docker Desktop
等待图标显示"运行中"
```

### 问题2：找不到 OpenClaw 镜像

**检查：**
```powershell
docker images | findstr openclaw
```

**解决：**
```powershell
# 方法1：重新标记镜像
docker tag <您的镜像> openclaw:latest

# 方法2：编辑配置
notepad .env
# 修改 OPENCLAW_IMAGE=您的镜像名
```

### 问题3：端口被占用

**检查：**
```powershell
netstat -ano | findstr :3000
netstat -ano | findstr :3001
```

**解决：**
```powershell
# 结束占用进程（替换<PID>）
taskkill /PID <PID> /F

# 或修改 .env 文件
PORT=3002
CLIENT_PORT=3001
```

### 问题4：权限问题

**解决：**
```powershell
# 右键点击脚本
# 选择"以管理员身份运行"
```
